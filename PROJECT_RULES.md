# PROJECT_RULES

部署名: 【制作部】_SES

## 目的
SES事業における案件情報・人材情報・スキルシート・メール情報を整理し、案件と人材のマッチング、提案候補作成、営業進捗管理を支援するアプリを制作する。

## 担当者
- PM担当: PMのいさお
- SES業務設計担当: 業務設計のひらおか
- 開発担当: 開発のほくろさん
- AIマッチング担当: マッチングのM
- セキュリティ担当: セキュリティのすーさん
- UI/運用設計担当: 運用設計のうんさん

## 運用方式
- PMハブ＋共有台帳方式で進める。
- 前田社長は原則PMのいさおへ依頼する。
- 各担当者の通常返答先はPMのいさお。
- 前田社長を担当者間の伝言係にしない。
- PMは依頼、返答回収、台帳反映、前田社長への要点報告を担当する。

## 中リスク上限つき自動許可

止まりすぎず、低〜中リスクの作業はPM判断で前に進める。

### 自動で進めてよい範囲
- ローカルHTML/CSS/JS作成・修正
- ダミーデータのみのプロトタイプ改善
- ローカル表示確認
- 構文チェック
- TASKS / STATUS / README更新
- 品質ゲート内の差し戻し修正
- 実データなし、外部送信なし、API接続なしの作業

### 今まで通り明示承認が必要な範囲
- 実データ、個人情報、機密情報本文の投入
- API接続
- 外部送信
- push / remote変更
- build
- 公開URL化
- ファイル削除
- 認証済みChrome操作
- 課金、購入、申込、送信操作

### 運用
- 中リスク上限内では、作業前に目的と範囲を短く固定して進める。
- 作業後に変更点、確認結果、禁止操作をしていないことを短く報告する。
- 不明な場合は安全側に倒して停止し、コピー用の承認文をコードブロックで出す。

## 扱う情報
- SES案件メール
- 人材情報
- スキルシート
- 単価、商流、稼働時期、面談情報
- 取引先会社情報
- 個人情報、連絡先、経歴情報

## 初期禁止事項
前田社長の明示承認なしに以下をしない。
- メール本文の取り込み
- スキルシート本文の読み込み
- 個人情報の記録
- 外部送信
- API接続
- Gmail / Google Workspace / Chatwork / LINE 連携
- ファイル削除
- build
- push
- remote変更
- 実行操作
- 顧客・取引先への送信

## 初期方針
まずは実データを使わず、SES業務フロー、情報項目、画面構成、マッチング基準、セキュリティ境界を整理する。
実メール・実スキルシート・実案件情報は、別途明示承認後に扱う。
