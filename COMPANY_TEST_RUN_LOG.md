# 企業テスト 実施ログ

## 目的

企業テストを複数社へ依頼した時に、送付状況、返却状況、PM判定、次アクションを追跡するための台帳。

## 実施ログ

| No | 依頼日 | 企業名 | 依頼先 | 共有方法 | 返却日 | 回収物 | PM判定 | 次アクション | 状態 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 1 |  | 匿名A社 | 役割のみ | ローカル / 公開URL |  | レポート / コメント / 回収票 | 継続 / 修正 / AI検討 / 本番連携 |  | 未依頼 / 依頼済 / 回収済 / 反映済 |
| 2 |  | 匿名B社 | 役割のみ | ローカル / 公開URL |  | レポート / コメント / 回収票 | 継続 / 修正 / AI検討 / 本番連携 |  | 未依頼 / 依頼済 / 回収済 / 反映済 |

## 記録ルール

- 企業名は匿名または仮名で記録する。
- 個人名、実メールアドレス、電話番号は記録しない。
- 実メール本文、実スキルシート本文は貼らない。
- コメントは要約だけ記録する。
- 詳細は `COMPANY_TEST_TRIAGE.md` に仕分ける。

## 状態の定義

| 状態 | 意味 |
| --- | --- |
| 未依頼 | まだ企業へ案内していない |
| 依頼済 | 案内文を渡した |
| 回収済 | 結果レポート、コメント、回収票のいずれかを受け取った |
| 反映済 | フィードバックをTASKS、項目名辞書、マッチング条件などへ反映した |

## PM報告用

```text
現在地:
企業テスト [依頼済] 社、[回収済] 社、[反映済] 社。

次にやること:
- 回収済みコメントを整理表へ反映
- 即修正分をローカルMVPへ反映
- 本番連携が必要なものは承認文を出す
```
