# 企業テスト サンプル結果レポート

## 目的

企業テスト後に、企業側からどのような形で結果を返してもらうかを事前に共有するためのサンプル。

## サンプル

```text
SES Auto Send テスト結果
マッチング点数: 85点 / 1位
提案候補基準: 80点以上 / 上位3名
判定人材数: 1名
提案候補人材: 1名
送信可能: 2件
除外: 1件

候補人材ランキング:
1. Javaバックエンドエンジニア: 85点 / 1位

送信可能企業:
- 株式会社サンプル / tanaka@example.invalid
- 株式会社停止先サンプル / sato@example.invalid

除外企業:
- 株式会社NG先サンプル: NGワード

PM判定:
本番連携設計へ進む: 点数、除外理由、メール文面の確認が揃っています。
次アクション: Gmail/API/DB連携の承認範囲を整理する

企業側コメント:
点数は概ね納得。単価NGと商流条件の表記ゆれをもう少し増やしたい。

企業側確認:
- 点数の納得感: 確認済み
- 除外理由: 確認済み
- メール文面: 確認済み
```

## PMが見るポイント

- 点数が営業感覚と大きくズレていないか
- 除外理由が納得できるか
- 提案メール文面が営業で使えそうか
- 追加したいNG条件や表記ゆれがあるか
- 本番連携に進めるか、ルール修正が必要か
