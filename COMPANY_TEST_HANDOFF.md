# 企業テスト手順

## 目的
企業にSES Auto SendのMVPを触ってもらい、営業感覚に合うか確認する。

## 使う画面

```text
http://127.0.0.1:4173/prototype/company-test.html
```

共有URLで渡す場合は、GitHub Pages反映が必要。push/公開URL化は明示承認が必要。

## 企業に依頼する確認
1. 案件情報、人材情報、送信先CSVを貼る。
2. `マッチング実行` を押す。
3. 点数、送信可能企業、除外理由、提案メール文面を見る。
4. コメント欄に違和感を書く。
5. 点数、除外理由、メール文面を確認済みにする。
6. 回収準備OKになったら結果レポートを保存する。

## 合格ライン

点数、除外理由、メール文面の3つが営業感覚と大きくズレていなければ、企業テストは継続可能。
大きなズレがある場合は、ルールベース修正またはAIプラン検討に回す。

## 見てほしい項目
- スキル一致
- 不足スキル
- 単価
- 勤務地/リモート
- 年齢制限
- 商流制限
- NGワード
- 送信停止
- 顧客別NG条件
- 提案メールの自然さ

## 企業へ伝える注意
- 実メール送信はしない。
- Gmail連携はしない。
- 外部API接続はしない。
- 実スキルシート本文は入れない。
- 個人情報は入れない。
- テスト用のダミーまたは匿名化した内容で確認する。

## 回収するもの
- テスト結果レポート
- 企業側コメント
- 点数の納得感
- 除外理由の違和感
- メール文面の違和感
- 追加したいNG条件
- 追加したい項目

## PMの判定

| 判定 | 条件 |
| --- | --- |
| 継続実装 | 点数、除外理由、メール文面が大きく外れていない |
| 修正必要 | 除外理由や点数に営業感覚とのズレがある |
| AI検討 | ルールベースでは案件/人材の読み取り精度が足りない |
| 本番連携検討 | テスト評価後にGmail/API/DBの承認を取る |

## テスト後に進めること
1. フィードバックを項目名辞書へ反映する。
2. NG条件の表記ゆれを追加する。
3. 必要ならAIプランの要否を判断する。
4. 本番連携が必要なら承認文を出す。
