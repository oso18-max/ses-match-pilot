# 企業テスト フィードバック整理表

## 目的

企業テスト後に戻ってきたコメントを、次の実装判断へすぐ変換するためのPM用整理表。

## 仕分け基準

| 区分 | 判断基準 | 次アクション |
| --- | --- | --- |
| 即修正 | 表記ゆれ、NG条件、点数配分などルールベースで直せる | 項目名辞書、マッチング条件、除外理由へ反映 |
| 後回し | MVP企業テストに直接影響しない管理項目や画面改善 | TASKSへ追加して優先度を下げる |
| AI検討 | 文章理解、画像PDF、曖昧な案件/人材判定などルールだけでは厳しい | AIプラン候補として費用影響を整理 |
| 本番連携 | Gmail、DB、認証、外部共有URL、送信実行が必要 | 承認文を作り、前田社長の明示承認後に進める |
| 却下 | 実データ投入、個人情報保存、無承認送信など安全境界に反する | 実装しない。代替案を検討 |

| 内容 | 担当 |
