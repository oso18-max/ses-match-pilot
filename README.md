# 【制作部】_SES

SES事業向けの案件・人材マッチング支援アプリ制作部署です。

## 目的
大量に届くSES案件メール・人材情報・スキルシートを整理し、案件と人材のマッチング、提案候補作成、営業進捗管理を支援する。

## 初期担当
- PM担当: PMのいさお
- SES業務設計担当: 業務設計のひらおか
- 開発担当: 開発のほくろさん
- AIマッチング担当: マッチングのM
- セキュリティ担当: セキュリティのすーさん
- UI/運用設計担当: 運用設計のうんさん

## 最初にやること
- SES業務フローを整理する
- 案件情報・人材情報・スキルシートの項目を整理する
- マッチング基準を整理する
- セキュリティ境界を決める
- 実データなしの画面・DB案を作る

## 現時点の制限
実メール、実スキルシート、個人情報、取引先情報、外部連携は未許可です。

## すぐテストする

先にローカルサーバーを起動する。

```text
START_COMPANY_TEST_SERVER.cmd
```

上のファイルを開いたままにしてから、ブラウザでURLを開く。

ローカルサーバー起動:

```powershell
node prototype\local-static-server.cjs
```

起動確認:

```text
http://127.0.0.1:4173/health
```

`ok` と表示されればローカルサーバーは起動中です。

普段開くトップページ:

```text
http://127.0.0.1:4173/prototype/index.html
```

元のトップUIはここです。最初に見る画面は `マッチング管理` です。
通常管理で見るのはこのトップページです。企業テストは左メニューには出さず、`テスト` 画面から必要な時だけ開きます。

企業テストをトップ内で開くURL:

```text
http://127.0.0.1:4173/prototype/index.html#companyTest
```

企業テストだけを直接開く専用ページ:

```text
http://127.0.0.1:4173/prototype/company-test.html
```

専用ページはトップではなく、企業テストだけを切り出した確認用ページ。
この専用ページでは、メール送信、Gmail連携、外部API接続、実データ保存は行わない。

画面で見る順番:
- `マッチング管理`
- `テスト`
- `企業テスト`
- `未送信確認`
- `疑似送信`
- `送信履歴`
- `返信検知`
- `面談管理`
- `成約管理`

企業に試してもらう場合:
- `企業テスト` を開く
- 案件情報を貼る
- 人材情報を貼る
- 複数人材を貼る場合は、人材ごとに `---` で区切る
- 送信先CSVを貼る
- 必要に応じて `提案候補の最低点` と `表示する上位人材数` を調整する
- `マッチング実行` を押す
- 実行前に案件スキル、人材スキル、送信先CSV、メール形式の準備状態を見る
- `読み取り結果` で、案件・人材・送信先がどう解釈されたかを見る
- `候補人材ランキング` で、複数人材の中から上位候補を見る
- ランキングでは、最低点以上を `候補`、基準未満を `基準外` として確認できる
- 点数、送信可能企業、除外理由、提案メールプレビューを見る
- 点数の内訳で、スキル一致・不足・単価・勤務地・年齢・商流・NGワードを見る
- 除外理由サマリーで、送信停止やNG条件の件数を見る
- 案件条件や顧客CSVで、年齢上限、商流上限、NGワードも確認できる
- `70万以上NG`、`45歳以上NG`、`商流二次まで`、`外国籍不可` などの表記ゆれも一部判定できる
- 送信先CSVは `company, person, email, sendable` に加えて、`会社名, 担当者名, メールアドレス, 送信可否` などの日本語ヘッダーも使える
- 企業側コメントに、点数の納得感やメール文面への意見を残せる
- 点数、除外理由、メール文面を確認済みチェックで残せる
- コメントと3つの確認チェックが揃うと、回収準備OKが表示される
- 入力内容をJSON保存・読込できる
- 入力はブラウザ内に一時保存される
- 結果レポートはコピーまたはテキスト保存できる
- 直近のテスト履歴をブラウザ内で確認できる
- テスト履歴にはコメント有無と確認チェック数も残る

結果の見方:
- 80点以上: 提案候補として確認
- 60〜79点: 要確認
- 59点以下: 見送り候補

注意:
- この段階では実メール送信しない
- Gmail接続しない
- API接続しない
- 実スキルシート本文は入れない
