# 【制作部】_SES

SES事業向けの案件・人材マッチング支援アプリ制作部署です。

## 目的
大量に届くSES案件メール・人材情報・スキルシートを整理し、案件と人材のマッチング、提案候補作成、営業進捗管理を支援する。

## 初期担当
- PM担当: PMのいさお
- SES業務設計担当: 業務設計のひらおか
- 開発担当: 開発のほくろさん
- AIマッチング担当: マッチングのM
- セキュリティ担当: セキュリティのすーさん
- UI/運用設計担当: 運用設計のうんさん

## 最初にやること
- SES業務フローを整理する
- 案件情報・人材情報・スキルシートの項目を整理する
- マッチング基準を整理する
- セキュリティ境界を決める
- 実データなしの画面・DB案を作る

## 現時点の制限
実メール、実スキルシート、個人情報、取引先情報、外部連携は未許可です。

## すぐテストする

ローカルサーバー起動:

```powershell
node prototype\local-static-server.cjs
```

ブラウザで開くURL:

```text
http://127.0.0.1:4173/prototype/index.html
```

企業テスト直リンク:

```text
http://127.0.0.1:4173/prototype/index.html#companyTest
```

企業テスト専用ページ:

```text
http://127.0.0.1:4173/prototype/company-test.html
```

企業へ渡す場合は、まず上記の専用ページを使う。

画面で見る順番:
- `テスト`
- `企業テスト`
- `未送信確認`
- `疑似送信`
- `送信履歴`
- `返信検知`

企業に試してもらう場合:
- `企業テスト` を開く
- 案件情報を貼る
- 人材情報を貼る
- 送信先CSVを貼る
- `マッチング実行` を押す
- 点数、送信可能企業、除外理由、提案メールプレビューを見る
- 入力はブラウザ内に一時保存される
- 結果レポートはコピーまたはテキスト保存できる
- 直近のテスト履歴をブラウザ内で確認できる

注意:
- この段階では実メール送信しない
- Gmail接続しない
- API接続しない
- 実スキルシート本文は入れない

自動テスト:

```powershell
node prototype\run-all-smoke-tests.cjs
```
