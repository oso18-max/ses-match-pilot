# SECURITY_REVIEW

## 初期セキュリティ方針

この部署は、SES案件、人材情報、スキルシート、取引先メール、単価、商流など機密性の高い情報を扱う可能性がある。

## 高リスク情報
- 個人名、連絡先、最寄駅、年齢、職務経歴
- スキルシート本文
- 所属会社、取引先会社、商流
- 単価、稼働条件、面談情報
- メール本文、メールアドレス、署名

## 初期禁止
- 実メール本文の取り込み
- 実スキルシート本文の読み込み
- 個人情報の記録
- 外部送信
- API接続
- Gmail / Google Workspace / Chatwork / LINE 連携
- 顧客・取引先への送信
- ファイル削除
- push

## 初期安全方針
まずはダミーデータ前提で、必要項目、マッチング基準、画面構成、権限管理、ログ方針を整理する。
実データを扱う場合は、前田社長の明示承認とセキュリティ確認を必須にする。
